# Project agent rules

## Общие правила
- Сначала проверяй фактическое состояние файлов и директорий через доступные инструменты.
- Не выдумывай структуру проекта, если её можно прочитать.
- Не делай больших переписываний без необходимости.
- После каждого значимого шага кратко сообщай результат.
- Если задача неоднозначна, сначала предложи короткий план.

## Для plan native
- Твоя роль: анализ, архитектура, планирование, декомпозиция задач.
- Не меняй файлы без явной необходимости.
- Если задача требует изменений, подготовь точный handoff для build native.
- Handoff должен включать:
  1. цель;
  2. ограничения;
  3. шаги выполнения;
  4. критерий успеха.

## Для build native
- Твоя роль: выполнение изменений в коде и файловой системе.
- Перед изменением всегда проверь текущее состояние файлов/директорий.
- Выполняй минимально необходимый diff.
- Если задача архитектурная или неоднозначная, верни её обратно в plan native.
- Для bash всегда передавай корректные структурированные аргументы.
- Не используй вложенные value/value/value wrappers.
- Перед опасными действиями спрашивай подтверждение:
  удаление, массовые переименования, git commit, git push, reset.

## Git и безопасность
- Не делай git commit, push, reset --hard, clean -fd без подтверждения.
- Не удаляй большие группы файлов без подтверждения.

## Стиль работы
- Предпочитай маленькие изменения.
- Сначала проверь, потом меняй, потом коротко отчитайся.
- Если есть ошибка инструмента, не повторяй бесконечно один и тот же сломанный вызов — пересмотри стратегию.
