# OpenCode Build Mode Workflow

## Activation

1. Load this file via `opencode.json` instructions
2. Confirm Build Mode is active
3. Read canonical rules: `/.agents/rules/programmer.md`

## Execution Flow

### Phase 1: Verify Handoff

- Read handoff from Plan Mode
- Understand goal, constraints, steps
- Ask clarifications if needed

### Phase 2: Pre-execution Check

- Verify workspace permissions
- Check existing files (read before write)
- Confirm spec exists and is approved

### Phase 3: Implementation

- Execute steps from handoff
- Minimal changes only
- Verify each step before next

### Phase 4: Validation

- Run tests if applicable
- Verify file contents
- Check no unintended changes

### Phase 5: Context Update

- Update `/.agents/memory-bank/active-context.md`
- Update `/.backlog/[task].md` with results
- Mark task complete

## Verification

- [ ] Implementation matches spec
- [ ] Tests pass (if applicable)
- [ ] Context updated
- [ ] Ready for QA/Human review
